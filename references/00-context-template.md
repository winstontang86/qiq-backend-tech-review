# 审查上下文模板

> 每次启动 review 前先填好此表，缺失项标 "未说明" 并向用户追问。

## 业务信息
- 业务重要性：{核心 / 重要 / 一般}
- 业务领域：{电商 / 金融 / 社交 / 内部工具 / ...}
- 用户规模：{DAU / MAU}
- 是否涉及资金：{是 / 否}

## 技术信息
- 团队规模：{X 人}
- 团队技术栈熟悉度：{列出团队熟悉的技术栈}
- 现有基础设施：{已有的中间件 / 平台}
- 期望上线时间：{X 周后}

## 系统规模
- 当前 QPS：{xxx}
- 预期 1 年后 QPS：{xxx}
- 当前数据量：{xxx}
- 预期 1 年后数据量：{xxx}

## SLA 要求
- 可用性目标：{99.9% / 99.95% / 99.99%}
- 核心接口延迟要求：{P99 < xxx ms}
- 数据一致性要求：{强一致 / 最终一致，最终一致的窗口}
- RPO：{xxx}（可容忍数据丢失时间：0 / 秒级 / 分钟级）
- RTO：{xxx}（可容忍恢复时间：秒级 / 分钟级 / 小时级）
- 部署模式：{单机房 / 同城双活 / 异地多活}

## 关键缺项追问

如果用户没提供以下任一项，必须追问后再进入审查：

1. 业务重要性等级
2. 是否涉及资金
3. 当前 QPS 量级
4. SLA 可用性目标
